# 12. Git Conventions

## 브랜치 전략

```
main (배포) ← dev ← feature/기능명 (작업 브랜치)
```

| 브랜치 | 용도 |
| --- | --- |
| `main` | 프로덕션 배포 |
| `dev` | `main` 머지 전 테스트/스테이징 (개발 버전) |
| `feature/*` | 기능 작업 — 완료 시 PR로 `dev`에 merge |
| `fix/*` | 버그 수정 — 완료 시 PR로 `dev`에 merge |
| `hotfix/*` | 긴급 수정 — 완료 시 PR로 `main`에 merge |

- 항상 **`dev`에서 브랜치를 생성**합니다.
- PR 대상은 **`dev`** 입니다.
- `dev` → `main`은 릴리즈 시 병합합니다.

### 브랜치 네이밍

1. **GitHub 이슈 기반** (대부분의 경우)
   - `<label>/#<issue번호>-<요약>` 형식
   ```
   feat/#123              # 기능 (이슈 #123)
   fix/#456               # 버그 수정 (이슈 #456)
   feat/#123-login-form
   ```
2. **이슈 없이 진행되는 작업**(hotfix 등)
   - `<type>/<요약>`

## 커밋 컨벤션

형식: `<type>(<scope>): <subject>`

- subject는 **명령조**로 짧게 작성.
- scope는 도메인/모듈명 (선택).

| type | 사용 시점 |
| --- | --- |
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 수정 (README, 주석 등) |
| `style` | 코드 포매팅, 세미콜론 등 (로직 변경 없음) |
| `refactor` | 동작 변경 없는 리팩터링 |
| `test` | 테스트 추가/수정 |
| `chore` | 빌드, 툴링, 패키지 변경 (프로덕션 코드 변경 없음) |
| `ci` | CI/CD 파이프라인 변경 |
| `comment` | 주석 추가/변경 |
| `HOTFIX` | 긴급한 수정 |

예시:

```
feat(auth): 로그인 폼 및 유효성 검사 추가
fix(user): 모바일에서 프로필 이미지 로드 안 되는 문제 수정
chore: tanstack-query v5.56.2로 업그레이드
docs(adr): ADR-0003 상태 관리 문서 추가
```

## PR 규칙

| 분류 | 규칙 |
| --- | --- |
| PR 단위 | PR당 최대 약 500줄 (초과 시 분할) |
| 승인 기준 | 최소 **2명 approve** 후 머지 가능 |
| 머지 권한 | approve를 받았다면 본인이 셀프 머지 허용 |
| 이슈 참조 | 필수 — `Closes #123` 또는 `Related to #123` |
| 새 의존성 | 추가 시 PR 설명에 도입 이유 명시 필수 |

### PR 템플릿

```md
## 📌 PR 제목

> PR 제목은 명확하게 작성해주세요.
> 예: [FEAT] 로그인 API 연동

---
## 🔗 관련 이슈 (Issue)

- Closes #이슈번호
- Related to #이슈번호

---
## ✅ 작업 내용

이번 PR에서 어떤 작업을 했는지 요약해주세요.

---
## 🖥️ 테스트 방법

리뷰어가 직접 확인할 수 있도록 테스트 방법을 적어주세요.

1. `/login` 페이지 접속
2. 아이디/비밀번호 입력
3. 로그인 버튼 클릭
4. 메인 페이지 이동 확인
```

## 이슈 템플릿

### ✨ Feature

```yaml
---
name: "✨ Feature"
about: 새로운 기능 추가
title: "[FEAT]"
labels: feature
assignees: ""
---

## ✅ 기능 설명
어떤 기능을 추가하려는지 간단히 설명해주세요.

---

## 🛠️ 작업 내용 (Todo)
- [ ]
- [ ]
- [ ]

---

## 🔗 참고 자료 (선택)
- 관련 문서:
- 참고 Issue:
```

### 🐛 Bug Report

```yaml
---
name: "🐛 Bug Report"
about: 버그 발생 시 제보하는 템플릿
title: "[BUG] "
labels: bug
assignees: ""
---

## 🐞 버그 설명
어떤 문제가 발생했는지 간단하고 명확하게 작성해주세요.

---
## 📌 재현 방법
1.
2.
3.
4.

---
## ✅ 기대한 동작
정상적으로 동작했을 경우 어떤 결과가 나와야 했는지 작성해주세요.

---
## ❗ 실제 동작
실제로는 어떤 일이 발생했는지 작성해주세요.

---
## 📷 참고 자료 (Screenshots / Logs)
- Screenshot:
- Console Error:

---
## 📝 추가 정보
예: 특정 사용자만 발생, API 응답 이상 등
```
